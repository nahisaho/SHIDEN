# はじめに

教育現場で働く皆さん、日々の授業準備お疲れ様です。

「授業計画を作りたいけど、ゼロから考えるのは大変...」
「ワークシートを作りたいけど、教育理論に基づいた質の高いものを作りたい」
「学習指導要領に準拠しているか毎回確認するのが手間...」

こんな悩みをお持ちではありませんか？

# 🤔 なぜ一般的なAIツールでは上手くいかないのか？

最近では **Microsoft 365 Copilot** や **Gemini**、**ChatGPT** など、様々なAIツールが利用可能になりました。しかし、教育コンテンツを作成しようとすると、こんな経験はありませんか？

```
👤 ユーザー: 授業計画を作成して
🤖 AI: どのような授業計画を作成しますか？対象の学年や教科を教えてください。

👤 ユーザー: 中学2年生の数学です
🤖 AI: どの単元についての授業計画ですか？

👤 ユーザー: 一次関数です
🤖 AI: 授業時間はどのくらいですか？

👤 ユーザー: 45分です
🤖 AI: 学習目標は何ですか？

... (延々と続く質問)

🤖 AI: では、以下の授業計画を作成しました！
（勝手に推論して、求めていない内容を出力し始める）
```

:::note warn
**なぜ上手くいかないのか？**

1. **コンテキストの蓄積がない** - 毎回ゼロから説明が必要
2. **教育専門知識がない** - Bloom's Taxonomyや教授法を知らない
3. **学習指導要領を参照しない** - 文科省の基準に準拠しているか不明
4. **質問が構造化されていない** - 何を聞くべきか分からず場当たり的
5. **出力フォーマットが不安定** - 毎回異なる形式で出力される
:::

結果として、**何度もやり取りを繰り返しても、欲しい品質のコンテンツにたどり着けない**という問題が発生します。

# 💡 SHIDENのアプローチ

SHIDENは、この問題を**4つの仕組み**で解決します。

| 課題 | SHIDENの解決策 |
|------|---------------|
| コンテキスト不足 | **1問1答の構造化ヒアリング**で必要情報を漏れなく収集 |
| 教育専門知識がない | **175件の教育理論DB**を内蔵し、自動で適用 |
| 学習指導要領未参照 | **2,469セクションの学習指導要領DB**を内蔵し、自動で引用 |
| 質問が場当たり的 | **メタプロンプト生成**で質問項目を事前定義 |
| 出力形式が不安定 | **スキル別テンプレート**で一貫したフォーマット |

**SHIDEN（師伝）** は、GitHub Copilotと連携して、教育者の皆さんの授業準備を支援するAgent Skillsパッケージです。

:::note info
**SHIDENの読み方**: 「しでん」
「師」から「伝」える、教育者を支援するという意味を込めています。
:::

# SHIDENの4つの特徴

# 1. 🎯 プロンプトの強化・構造化

曖昧なリクエストを自動的に構造化し、最適なコンテンツを生成します。

**Before（ユーザー入力）:**
```
授業計画を作成して
```

**After（SHIDENが強化）:**
```
📝 あなたのリクエストを以下のように理解しました：

【元のリクエスト】授業計画を作成して
【強化されたリクエスト】対象学年・教科・単元を指定した授業計画の作成
【補足した要素】
- Bloom's Taxonomyに基づく学習目標設定
- 時間配分と指導上の留意点
- 評価規準

👉 この理解で正しいですか？修正があればお知らせください。
```

# 2. 💬 対話を通じたコンテキスト強化（メタプロンプト生成）

1問1答形式で必要な情報を収集し、質の高いメタプロンプトを生成します。

```
📌 [Q1/5] この授業で、生徒に達成してほしい具体的な目標は何ですか？
（例: 「一次関数のグラフを正確に描けるようになる」）

💡 スキップする場合は「スキップ」とお伝えください。
```

:::note warn
**1問1答ルール**: SHIDENは複数の質問を一度に投げません。1つの質問 → 回答待ち → 次の質問、という流れで進行します。
:::

# 3. 📚 175件の教育理論に基づいたドキュメント作成

SHIDENは**175件の教育理論をSQLiteデータベースとして内蔵**しています。
生成されるコンテンツには、必ず教育理論的な根拠が付与されます。

| カテゴリ | 件数 | 例 |
|---------|------|-----|
| 学習理論 | 45 | 構成主義、行動主義、社会的構成主義 |
| アジアの教育理論 | 27 | 授業研究、学び合い |
| 授業設計 | 19 | Gagné's Nine Events、ADDIE |
| 教育工学 | 18 | SAMR Model、TPACK |
| 動機づけ | 10 | ARCS Model、自己決定理論 |
| 評価理論 | 10 | 形成的評価、ルーブリック |

**引用される主な理論:**
- **Bloom's Taxonomy** - 認知的領域の分類（学習目標設定に活用）
- **Gagné's Nine Events of Instruction** - 教授の9事象（授業展開に活用）
- **ARCS Model** - 動機づけモデル（教材設計に活用）
- **Zone of Proximal Development** - 最近接発達領域（個別指導に活用）
- **Growth Mindset** - 成長マインドセット（フィードバックに活用）

# 4. 📖 学習指導要領に基づくドキュメント作成（v0.4.0〜）

**文部科学省の学習指導要領（小学校・中学校・高等学校）をSQLiteデータベースとして内蔵**しています。FTS5（Full-Text Search）による日本語全文検索と、WITH RECURSIVE CTEによるセクション間の関連走査を実現し、Graph的なクエリが可能です。

| ドキュメント | セクション数 |
|-------------|-------------|
| 小学校学習指導要領（平成29年告示） | 538 |
| 中学校学習指導要領（平成29年告示） | 560 |
| 高等学校学習指導要領（平成30年告示） | 1,371 |
| **合計** | **2,469** |

生成されるコンテンツには、必ず「学習指導要領との関連」セクションが含まれます。

```markdown
# 学習指導要領との関連

**参照箇所**: 中学校学習指導要領 第2章 第3節 数学 第2学年
**目標**: 「一次関数について、数学的活動を通して...」
**内容**: 「一次関数のグラフの特徴を理解すること」
**本時との対応**: 本時では特に「グラフの読み取り」に焦点を当て...
```

# 対応できるドキュメントの種類

SHIDENは**6つのユースケーススキル**を提供しています。

| スキル | 説明 | リクエスト例 |
|--------|------|-------------|
| 📚 **授業計画作成** | Bloom's Taxonomy基準の指導案生成 | 「高1英語のスピーキング授業計画を」 |
| 📝 **教材作成** | ワークシート、スライド、クイズ生成 | 「小数の足し算のワークシートを」 |
| ✅ **評価設計** | ルーブリック、テスト問題生成 | 「プレゼンの5段階ルーブリックを」 |
| 👤 **個別指導計画** | 学習者特性に応じた戦略提案 | 「読み書きが苦手な生徒への対応を」 |
| 💬 **フィードバック** | Growth Mindsetに基づく振り返り支援 | 「この作文にフィードバックを」 |
| 🤝 **生活指導案** | 発達段階を考慮した指導案 | 「友人関係のトラブルへの対応を」 |

# ワークフロー（4フェーズ）

SHIDENは以下の4フェーズで教育コンテンツを生成します。

```mermaid
graph LR
    A[Phase 1: メタプロンプト生成] --> B[Phase 2: 学習指導要領参照]
    B --> C[Phase 3: スキル実行]
    C --> D[Phase 4: 理論引用・エビデンス付与]
```

| Phase | 内容 |
|-------|------|
| **Phase 1** | ユーザーとの1問1答でコンテキスト収集、メタプロンプト生成 |
| **Phase 2** | 学習指導要領DBを検索し、該当する目標・内容を確認 |
| **Phase 3** | 適切なスキルを選択し、コンテンツ生成 |
| **Phase 4** | 教育理論DBを検索し、理論的根拠を付与 |

**必須環境**

- **VS Code**: GitHub Copilot拡張機能がインストール済み

# インストール方法

# Step 1: インストール

```bash
npm install shiden
```

# Step 2: 初期化

```bash
npx shiden init
```

これにより、以下のファイルがプロジェクトにコピーされます。
- `AGENTS.md` - GitHub Copilot Agent Skillsエントリーポイント
- `.github/prompts/` - スキル定義ファイル（7種類）
- `.github/skills/` - 統合スキルファイル（3種類）

# Step 3: 使用開始

VS Codeでプロジェクトを開き、GitHub Copilot Chatで話しかけてください：

```
@workspace 中学2年生の数学で「一次関数」の授業計画を作成してください
```

# 内部で利用されるCLI（参考）

:::note info
**通常、CLIを直接操作する必要はありません**。GitHub Copilotとの自然言語でのやり取りの中で、SHIDENが内部的にこれらのコマンドを実行し、教育理論や学習指導要領を検索します。以下は仕組みを理解したい方向けの参考情報です。
:::

# 教育理論検索（内部処理）

```bash
# カテゴリ一覧
npx shiden theories categories

# 理論を検索（日本語対応）
npx shiden theories search "社会的構成主義"

# 理論の詳細を取得
npx shiden theories get theory-003

# 関連理論をグラフ走査
npx shiden theories related theory-003 -d 2
```

# 学習指導要領検索（内部処理）

```bash
# キーワード検索
npx shiden curriculum search "プログラミング"
npx shiden curriculum search "論理的思考"

# 教科別検索
npx shiden curriculum subject 算数
npx shiden curriculum subject 英語

# 学年別検索
npx shiden curriculum grade "第3学年"

# 統計表示
npx shiden curriculum stats
```

# 使用例

# 例1: 授業計画作成

**リクエスト:**
```
中学2年生の数学で「一次関数」の授業計画を作成してください。
45分授業で、グラフの読み取りを中心にしたいです。
```

**出力（抜粋）:**
```markdown
# 授業計画

# 基本情報
| 項目 | 内容 |
|------|------|
| 対象学年 | 中学2年生 |
| 教科 | 数学 |
| 単元名 | 一次関数 |
| 授業時間 | 45分 |

# 学習目標
- **認知レベル**: 応用（Apply）
- **目標文**: 生徒は、一次関数のグラフから傾きと切片を読み取り、式を求めることができる。

# 学習指導要領との関連
**参照箇所**: 中学校学習指導要領 第2章 第3節 数学 第2学年
**目標**: 「一次関数について、数学的活動を通して...」

# 参照した教育理論
- **Bloom's Taxonomy**: 学習目標を「応用」レベルに設定
- **Gagné's Nine Events**: 導入での「前提条件の想起」を活用
```

# 例2: 教材作成

**リクエスト:**
```
小学4年生の理科「電気の流れ」のワークシートを作成してください。
実験の手順と考察欄を含めてください。
```

# 例3: 評価設計

**リクエスト:**
```
高校1年生の英語プレゼンテーション用の5段階ルーブリックを作成してください。
発音、内容、構成、アイコンタクト、時間管理を評価項目にしてください。
```

# 注意事項

:::note warn
**生活指導における危機対応について**

生活指導において危機的状況（自傷、いじめ、虐待等）が疑われる場合は、必ず以下を含めます。

> ⚠️ **重要**: この状況については、スクールカウンセラー、管理職、または専門機関への相談を強くお勧めします。AIの提案はあくまで参考情報であり、専門家の判断に代わるものではありません。
:::

:::note info
**教育理論の限界について**

教育理論は万能ではありません。生成されたコンテンツは、教育者自身の専門的判断と組み合わせて活用してください。
:::
